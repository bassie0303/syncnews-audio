## UX / プレーヤー
- [ ] **表示/音声プリセット4種の1タップ切替**
  「表示言語 × 音声言語」の組み合わせを4プリセットとして1タップで巡回切替できるようにする:
  1. 日本語表示 × 日本語音声
  2. 英語表示 × 英語音声
  3. 日本語表示 × 英語音声
  4. 英語表示 × 日本語音声
  現状はテキスト/音声を独立トグル（2スイッチ）。これに加えて「よく使う4組」を素早く回せるUIを用意する。
  例: プリセットチップの横スクロール or 1ボタンで 1→2→3→4 を巡回。

- [ ] **記事ごとの再生形式（4パターン）選択 + 設定の引き継ぎ**
  - 各記事に対して、上記4パターンのどれで再生/表示するかを選べる。
  - **次に再生・表示する記事は、直前に選んだ設定を引き継ぐ**（最後に使ったプリセットが既定になる）。
  - 実装メモ: 「現在の選好プリセット」をアプリ状態として保持（永続化も検討）。プレーヤーを開くとき
    その選好を初期値に。記事個別に上書きした場合の保持範囲（その記事だけ／以降の既定も更新）を要設計。
    上の「プリセット4種の1タップ切替」と一体で考える。

## 音声品質
- [ ] **日本語の漢字読み精度の改善**
  ElevenLabs（多言語モデル）は漢字の読み・固有名詞・数字を誤読しがち。対策候補:
  - 日本語トラックを日本語特化TTS（Google Cloud TTS / Azure / VOICEVOX 等、タイムスタンプ取得可能なもの）に出し分け
  - もしくは形態素解析で読み（カナ）に変換してから合成（※表示は漢字のまま・文単位で対応付け）
  - ElevenLabs 発音辞書で頻出固有名詞を登録
  ※ 本項目は別途着手中（形態素解析→カナ化アプローチ）。

## 同期精度
- [ ] **日↔英の文数1:1保証**
  翻訳がGPTで文を統合/分割すると日英の文数がズレ（例 ja32/en44）、言語切替時の `mapPositionForLanguageSwitch` が相対進捗フォールバックになる。文単位翻訳（原文を文分割してから各文を翻訳）で1:1を保証する。

## 記事管理 / 一覧
- [ ] **履歴（記事）の削除** — 一覧から記事を削除（DB行＋Storage音声も合わせて削除）。スワイプ削除 or 長押しメニュー。
