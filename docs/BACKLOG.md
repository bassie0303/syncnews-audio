## UX / プレーヤー
- [x] **表示/音声プリセット4種の1タップ切替**
  「表示言語 × 音声言語」の4組（日×日 / 英×英 / 日×英 / 英×日）をプレーヤー下部のプリセットチップ（📄表示 / 🔊音声）で1タップ適用。既存の独立トグルと併存し、トグルもプリセット選択に追従する。`_applyPreset()` が既存の `_switchTextLang`/`_switchAudioLang` に委譲（差分のみ反映・現在地マッピング維持）。

- [ ] **記事ごとの再生形式（4パターン）選択 + 設定の引き継ぎ**
  - 各記事に対して、上記4パターンのどれで再生/表示するかを選べる。
  - **次に再生・表示する記事は、直前に選んだ設定を引き継ぐ**（最後に使ったプリセットが既定になる）。
  - 実装メモ: 「現在の選好プリセット」をアプリ状態として保持（永続化も検討）。プレーヤーを開くとき
    その選好を初期値に。記事個別に上書きした場合の保持範囲（その記事だけ／以降の既定も更新）を要設計。
    上の「プリセット4種の1タップ切替」と一体で考える。

- [ ] **最初に戻るボタン** — 再生位置を記事の先頭（最初のセグメント）に一発で戻すボタン。プレーヤーのコントロールに追加（`audio.seek(Duration.zero)` ＋ index リセット）。

- [x] **リピート機能（記事全体）** — プレーヤーのリピートトグルで記事全体を繰り返し再生。`LoopMode.one` は setUrl 単一音源だと環境により無効なため、`processingState==completed` を監視して先頭へ seek+play する方式。一巡の区切りが分かるよう**約1.3秒の無音の間**を挿入。言語/速度切替後も維持。将来: 文単位リピート（今の文を数回）・区間リピート（A-B）は未実装。

- [ ] **スリープタイマー機能** — 「15分後／30分後／再生中の記事の最後まで」などで自動停止。就寝前のリスニング用。タイマー残り時間の表示と、終了時のフェードアウトも検討。

## 音声品質
- [ ] **日本語の漢字読み精度の改善**
  ElevenLabs（多言語モデル）は漢字の読み・固有名詞・数字を誤読しがち。対策候補:
  - 日本語トラックを日本語特化TTS（Google Cloud TTS / Azure / VOICEVOX 等、タイムスタンプ取得可能なもの）に出し分け
  - もしくは形態素解析で読み（カナ）に変換してから合成（※表示は漢字のまま・文単位で対応付け）
  - ElevenLabs 発音辞書で頻出固有名詞を登録
  ※ 本項目は別途着手中（形態素解析→カナ化アプローチ）。

- [~] **日本語音声の不自然な区切り（音引き・単語途中での分割）** — 調査の結果、`_to_reading_ja` の読みが**カタカナ**で、合成テキストが「カタカナ内容語＋ひらがな助詞＋数字」の混在になり、カタカナ過多でTTSの韻律が崩れて語の途中/長音符でつっかえていたのが主因と判断。
  対策: 読みを**ひらがな**で返すよう変更（`_kata_to_hira`）。本来のカタカナ語（サーバー等）は別トークンで原文のまま残り、全体が自然なかな文に近づく。音は同じ・低リスク。→ **実機での音声確認待ち**（要 ElevenLabs クレジット。短い記事なら現残量でも可）。
  残課題: 数字＋かなの境界（例「15にち」）の読み・固有名詞、`スカイツリー`等がひらがな化される点は許容。さらなる自然さは日本語特化TTSへの出し分けが本筋。

### 再生の安定性（要調査・関連しうる一群）
以下は同根の可能性あり（音声出力/フォーカス/バックグラウンドサービス周り）。再現条件のログ取りから着手する。
**根本対策を投入済み**: `audio_session` が依存にあるのに `AudioSession.configure()` 未呼び出しだった（フォーカス・割り込み・ダッキング・ヘッドホン抜けが全て未制御）。
`SyncAudioHandler.configureSession()` を追加し `initAudioService()` で初期化時に実行。割り込みの一時停止→自動再開、ダッキング→音量復帰、becomingNoisy→一時停止、トラックロード時の音量リセットを実装。→ **実機での再確認待ち**。
- [~] **英語音声が極端に小さくなることがある** — ダッキングからの音量復帰＋トラックロード時の `setVolume(1.0)` で「減音のまま固定」は解消見込み。なお残課題として **生成音声（ElevenLabs）側の日英ラウドネス差**は別要因として残る（生成 or 再生時のゲイン正規化を要検討）。
- [~] **音が出ないことがある（起動時・再生開始時・途中で消える）** — オーディオフォーカス未取得が主因と推定。`AudioSession.configure()` 実装で改善見込み。実機で再発するかを確認。
- [~] **バックグラウンド再生が停止することがある** — 実機確認で「通知は残るが音だけ止まる」と判明。`WAKE_LOCK` 権限が抜けており、画面オフ時にCPU/WiFiが落ちてネットワーク音源のストリームが停止していたのが主因と推定。`AndroidManifest` と `setup_native.sh` に `WAKE_LOCK` を追加。割り込み後の自動再開も実装済み。残課題: OEMのバッテリー最適化由来のkillは端末側ホワイトリスト設定が必要な場合あり（`docs/native_setup.md`）。将来的には音源のローカルダウンロード再生でネットワーク依存を断つ案も。
