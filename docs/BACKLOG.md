## UX / プレーヤー
- [x] **表示/音声プリセット4種の1タップ切替**
  「表示言語 × 音声言語」の4組（日×日 / 英×英 / 日×英 / 英×日）をプレーヤー下部のプリセットチップ（📄表示 / 🔊音声）で1タップ適用。既存の独立トグルと併存し、トグルもプリセット選択に追従する。`_applyPreset()` が既存の `_switchTextLang`/`_switchAudioLang` に委譲（差分のみ反映・現在地マッピング維持）。

- [x] **記事ごとの再生形式（4パターン）選択 + 設定の引き継ぎ** — `PlaybackPrefs` で実装。
  - 引き継ぎ: 直近の「表示×音声」を次に開く記事の既定に（`textLang`/`audioLang`）。
  - 記事個別: `perArticle[articleId]` に記録し、その記事を開き直したら優先復元（`initialFor()`）。
  - [ ] 残: **永続化**（アプリ再起動後も保持、SharedPreferences）。現状はアプリ起動中のみ保持。

- [x] **最初に戻るボタン** — トランスポートに ⏮ を追加。`audio.seek(Duration.zero)` で先頭へ（同期ハイライトも追従）。

- [x] **リピート機能（記事全体）** — プレーヤーのリピートトグルで記事全体を繰り返し再生。`LoopMode.one` は setUrl 単一音源だと環境により無効なため、`processingState==completed` を監視して先頭へ seek+play する方式。一巡の区切りが分かるよう**約1.3秒の無音の間**を挿入。言語/速度切替後も維持。将来: 文単位リピート（今の文を数回）・区間リピート（A-B）は未実装。

- [x] **スリープタイマー機能** — AppBar右上の 🌙 メニューで「15/30/45/60分後／この記事の最後まで／オフ」。固定分は壁時計タイマーで残り時間を mm:ss 表示し0で `audio.pause()`。「この記事の最後まで」は完了イベントで停止する `setStopAtEnd` 方式に変更し、**リピートより優先**で停止（再生速度にも左右されない）。将来: 終了時フェードアウトは未実装。

## 既知の不具合
- [x] **一覧がアプリ復帰時に空になることがある** — ストリームをフィールド保持し、`resumed` で再購読＋一発 `fetchList()` で即補完。直前の一覧を保持して再購読中の一時的な空emitでも表示維持。プルリフレッシュ（RefreshIndicator）も追加。

### 再生の安定性（対策完了・実機確認済み）
同根（音声出力/フォーカス/バックグラウンドサービス周り）として一括対応。**根本原因**: `audio_session` が依存にあるのに `AudioSession.configure()` 未呼び出しで、フォーカス・割り込み・ダッキング・ヘッドホン抜けが全て未制御だった。加えて `WAKE_LOCK` 権限が欠落していた。
**対策（全て配信APKに焼込み確認済み・aapt2で権限検証）**: `SyncAudioHandler.configureSession()` を `initAudioService()` で実行（フォーカス取得・割り込み一時停止→自動再開・becomingNoisy→一時停止・トラックロード時 `setVolume(1.0)`）。ダッキングは自前で下げず終了時に1.0へ復帰のみ。`WAKE_LOCK` を `AndroidManifest`/`setup_native.sh` に追加。
- [x] **英語音声が極端に小さくなる** — 解消（実機OK）。自前ダッキング撤廃＋音量復帰で「減音のまま固定」をなくした。残: ElevenLabs生成音声の日英ラウドネス差は別要因（必要なら将来ゲイン正規化）。
- [x] **音が出ない（起動時・再生開始時・途中で消える）** — 解消（実機OK）。オーディオフォーカス未取得が主因で `AudioSession.configure()` で解決。
- [x] **バックグラウンド再生が停止する** — 解消（実機OK）。「通知は残るが音だけ止まる」＝`WAKE_LOCK` 欠落で画面オフ時にストリーム停止していた。権限追加＋割り込み後自動再開で対応。
  - 残課題（任意）: OEMのバッテリー最適化由来のkillは端末側ホワイトリスト設定が必要な場合あり（`docs/native_setup.md`）。「時々音量が下がる」が残る場合はOSレベルの一時ダッキング（自動復帰）で、アプリ側の固定減音ではない。将来オフライン再生（音源ローカルDL）でネットワーク依存を断つ案も。
